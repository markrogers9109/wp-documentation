I've already explained how uploads and plugins work with our workflow currently, so I should also explain database and themes before you get started

Like I've said before, you should be keeping one thing in your mind whenever you're working on one of these Docker apps: what's gonna happen to this site if I deploy it on a brand new server?

With uploads and plugins, you just need to add your changes to the repo

The database is similar, but it trickier to handle since we don't have a good way for you to get your own database backup from the live site

So I typically handle that during the review process: once a merge request has been accepted, I'll merge to develop then push a fresh DB dump (heh) straight into develop

What that means for you is I'll be asking you if your merge request has any database changes, and if it doesn't, I'll probably save myself the effort

Finally, changing the theme requires an environment variable be changed for deployment purposes, so if you switch to a new theme, you'll need to inform me in your merge request that a new theme is being used
